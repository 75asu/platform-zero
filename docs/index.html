platform-zero

// production infrastructure. one command.

A self-contained SRE platform kit that runs on a single Linux machine with SSH access. No cloud account. No managed services. Real tools, real patterns, real understanding.

// labs

// foundation - runs standalone

awslab live

AWS infrastructure - Terraform modules, Terragrunt multi-env, 11 modules across dev + staging. Runs against Ministack locally.

k8slab in progress

Kubernetes platform - k3s, ArgoCD app-of-apps, Vault + ESO, Gitea CI, full LGTM stack. GitOps from day one.

gcplab live

GCP infrastructure - 6 modules (IAM, GCS, Pub/Sub, Cloud SQL, Secret Manager, BigQuery) across dev + staging. Terraform + Terragrunt against MiniSky emulator via nginx proxy.

dblab planned

Database operations - PostgreSQL HA with Patroni, PgBouncer, WAL-G backups, PITR testing, CDC with Debezium, fire drills.

advanced layers - depend on foundation above

// advanced - layers on top of foundation

networkinglab planned

Networking internals - eBPF packet tracing, WireGuard from scratch, BGP with FRR, Cilium network policies, Linkerd mTLS dissection.

securitylab planned

Supply chain and runtime - Trivy/Grype scanning, Cosign image signing, SBOM attestations, Falco runtime rules, OPA/Rego policies, CIS benchmarks.

platformlab planned

Internal developer platform - Backstage IDP, service catalog wired to awslab + k8slab, golden path templates, Crossplane self-service infra.

// quick start
01

Prerequisites

A Linux machine with SSH access, 4GB+ RAM. Ansible, kubectl, and git on your local machine.

02

Configure

cp .env.example .env
# fill in TARGET_HOST, TARGET_USER, SSH_KEY_PATH
03

Run

make up      # blank machine → full platform (~10 min)
make down    # full teardown, machine is clean
// the gitops rule
Ansible → OS layer only (k3s, kubeconfig)
kubectl → Bootstrap only. Runs once, ever.
git push → Everything else. This is the only way to change cluster state.
